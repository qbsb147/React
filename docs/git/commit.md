# 커밋 규칙

## 기본 기능 구현시

- 제목은 어떤 페이지에서
- 내용은 어떤 기능을 추가했다는 형식으로 작성합니다.

### 제목 형식

- 특히 제목에서 한글을 사용하며 URL, 파일명과 같은 것을 제목으로 하지 않습니다.
- 제목은 한 두단어로 이뤄지며 `대시보드, 퍼널 필터, 사이드바`와 같이 간략하게 적습니다.

## 불안정한 파일을 커밋할 시

버그가 있거나 보완이 필요하지만 급하게 commit을 해야하는 경우 내용이 길어지는 것이 불필요한 경우가 많으므로  
`warn!`를 앞에 앞에 붙이고 뒤에 참고해야할 내용을 간략히 작성합니다.

## 앞의 커밋 내용과 중복

- 해당 기능에 대한 구현 및 커밋을 이미 했지만 뒤늦게 버그가 발견되거나 수정 및 반영이 완성되지 않은 것이 발견된 경우  
  앞에서 커밋한 내용에서 벗어나지 않은 내용이라면  
  `[fix]`를 앞에 붙이고 앞선 커밋의 해당 제목만 뒤에 작성해줍니다.

## 팀원이 참고해야할 문서가 올라왔을 시

`[docs]`를 앞에 붙이고 파일명만 간단히 뒤에 작성합니다.

## 빠른 수정 및 보완이 필요할 시

`!!!`를 앞에 붙이고 간략히 내용을 작성합니다.

## 만약 어떠한 고민을 하다가 작업을 끝낸다면.

만약 필요하다면 작성하며 `'#'`를 양 끝에 붙이고 해당 내용을 작성합니다.

## 다른 브랜치에서 해야할 작업을 해당 브랜치에서 작업했을 시

필요에 의해 혹은 의도치 않게 다른 브랜치 작업을 해당 브랜치에서 작업했을 경우 충돌 가능성이 높습니다.  
그럴 경우를 대비해 다른 브랜치 명을 `(브랜치명)`을 내용 혹은 제목 앞에 붙여줍니다.
